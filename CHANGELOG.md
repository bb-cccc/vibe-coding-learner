# Changelog

本文件遵循 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/) 格式。

## [2.2.0] — 2026-06-27

### 移除

- **输入源从四种缩减为两种**：删除"项目分析模式"和"组合模式"。这两个模式从定义之初即为占位符（有名称、有一行描述，但无展开的执行逻辑）。对应的 README、examples、templates 中的引用已同步清理。

### 新增

- 排错维度下新增可选提示：用户如果手头有相关的 git diff 或 commit，可以在复盘时手动贴入辅助还原，但 Skill 不会主动扫描代码仓库。

### 变更理由

留着空壳模式产生了两个问题：误导用户以为可用；持续诱发"什么时候把 git 分析做完"的建议。实际上 vibe coding 场景下 git commit 质量普遍极低，自动从中提取知识是不可靠的——Agent 对话才是真正包含"为什么这么选、怎么排查、踩了什么坑"的信息源。精力应该集中在对话分析这个单一壁垒上，而不是摊开做四个半吊子模式。

## [2.1.0] — 初始发布

### 核心功能

- 三种处理模式：拦截记录（5 维度 × 6 要素知识提取）、面试题生成（10 类题型 + 反事实推理 + 复述强制）、知识蒸馏（6 个持久化文件 + SM-2 间隔复习）
- 9 道防御机制：用户确认、范围限制、复述强制、渐进式提醒、信号过滤、深度标注、去重合并、过期审查、冷启动引导
- 难度自适应：根据用户经验等级（初级/中级/高级）调整题目分布
- 知识衰减管理：90/180/365 天审查触发 + 归档机制
