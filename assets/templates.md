# 输出模板

本文件定义了 vibe-coding-learner 所有输出内容的标准格式模板。

---

## 模板一：面试题卡片

```markdown
### [类型] {题目名称}

**难度**：{基础 | 进阶 | 高级}
**标签**：#{标签1} #{标签2} #{标签3}
**来源**：基于 {项目名} 的 {阶段} 阶段提取
**生成日期**：{YYYY-MM-DD}

#### 场景描述

{用 2-3 句话描述具体场景，要求有代入感，让答题者仿佛身处真实工作环境}

#### 问题

{明确的面试问题，一句话}

#### 参考思路

{不是标准答案，而是"应该怎么思考这个问题"的步骤}

1. **首先**：{分析问题的切入点}
2. **然后**：{需要考虑的关键因素}
3. **接着**：{方案设计的核心逻辑}
4. **最后**：{如何验证和总结}

#### 参考答案要点

{面试时应该提到的核心内容，用要点列出}

- {要点 1}
- {要点 2}
- {要点 3}

#### 反事实推理

{改变约束条件，考察是否真正理解}

- 如果把 {技术 A} 换成 {技术 B}，方案需要怎么调整？
- 如果规模扩大 {N} 倍，瓶颈会出现在哪里？
- 如果团队只有 2 个人，方案需要怎么简化？

#### 深挖追问

{面试官可能的追问，帮助准备更深层回答}

- {追问 1}
- {追问 2}
- {追问 3}

#### 学习状态

- **掌握度**：未复述 / 0-完全卡住 / 1-部分回忆 / 2-流畅复述
- **上次复述日期**：{YYYY-MM-DD}
- **卡住位置诊断**：无 / 概念不清 / 逻辑链断裂 / 细节遗忘
- **下次复习日期**：{YYYY-MM-DD}（当前间隔：{N} 天）
- **归档状态**：活跃 / 已过时 / 已替代 / 仍有效
```

## 模板二：避坑条目

```markdown
### [坑-{编号}] {简短标题}

**领域**：{编码 | 部署 | 架构 | 调试 | 运维}
**严重程度**：{低 | 中 | 高 | 致命}
**来源**：{项目名} - {日期}

#### 现象

{描述问题的外在表现，看到了什么？报了什么错？}

#### 根本原因

{为什么会出这个问题？底层原因是什么？}

#### 解决过程

{当时是怎么排查和解决的？关键转折点是什么？}

1. {步骤 1}
2. {步骤 2}
3. {步骤 3}

#### 解决方法

{最终的解决方案是什么？}

#### 预防措施

{以后怎么避免再踩同样的坑？}

- {措施 1}
- {措施 2}

#### 关联知识点

- 关联面试题：{Q-编号}
- 关联知识条目：{KB-编号}
```

## 模板三：知识库条目

```markdown
### [KB-{编号}] {知识标题}

**领域**：{架构决策 | 代码实现 | 排错方法 | 部署运维 | 方法论}
**深度**：{基础 | 进阶 | 高级}
**来源**：{项目名} - {阶段} - {日期}
**标签**：#{标签1} #{标签2}
**创建日期**：{YYYY-MM-DD}
**技术栈版本**：{如 Python 3.11 / React 18 / Docker 24}
**归档状态**：活跃 / 已过时 / 已替代 / 仍有效
**最后审查日期**：{YYYY-MM-DD}

#### 概述

{一段话概括这个知识点的核心内容}

#### 背景

{什么情况下需要这个知识？解决什么问题？}

#### 核心内容

{详细解释，从第一性原理出发，不只是"怎么做"，更要解释"为什么"}

#### 代码示例（如适用）

```{语言}
// 带注释的代码示例
// 每段关键代码都要有解释
```

#### 常见误区

- {误区 1}：{为什么是错的}
- {误区 2}：{为什么是错的}

#### 适用条件与限制

{这个知识在什么条件下适用？有什么限制？什么情况下不适用？}

#### 关联

- 相关面试题：{Q-编号列表}
- 相关避坑条目：{坑-编号列表}
- 相关知识条目：{KB-编号列表}
```

## 模板四附加：复习调度卡片

```markdown
### {Q-编号 或 KB-编号}：{标题}

- **类型**：面试题 / 知识条目
- **创建日期**：{YYYY-MM-DD}
- **复习历史**：

| 日期 | 评分（0/1/2） | 卡住诊断 | 间隔变化 |
|------|--------------|----------|----------|
| 2026-06-24 | 1 — 部分回忆 | 细节遗忘 | 初始 → 1天 |
| 2026-06-25 | 2 — 流畅复述 | 无 | 1天 → 3天 |
| 2026-06-28 | 2 — 流畅复述 | 无 | 3天 → 7天 |

- **当前间隔**：{N} 天
- **下次复习日期**：{YYYY-MM-DD}
- **掌握状态**：🟢 已掌握 / 🟡 复习中 / 🔴 困难 / ⬜ 未复述
```

## 模板五：归档记录

```markdown
# 归档记录

> 最后更新：{YYYY-MM-DD}
> 总归档条目：{N} 条

## 归档日志

| 日期 | 编号 | 标题 | 原创建日期 | 归档原因 | 归档前状态 |
|------|------|------|-----------|----------|-----------|
| 2026-09-24 | Q-012 | AngularJS 双向绑定原理 | 2026-01-15 | 技术已过时（AngularJS EOL） | 仍有效 |
| 2026-09-24 | 坑-005 | Python 2.7 编码问题 | 2026-03-01 | 技术已过时（Python 2.7 EOL） | 已过时 |

## 按归档原因分类

### 已过时（{n} 条）
- [{编号}] {标题} — 归档日期：{日期}

### 已替代（{n} 条）
- [{编号}] {标题} — 替代方案：{新条目编号} — 归档日期：{日期}
```

## 模板六：归档条目头

用于标记已归档的面试题或知识条目，在条目开头插入：

```markdown
> **⚠️ 已归档 — {过时原因/替代说明}**
> 归档日期：{YYYY-MM-DD} | 此条目不再参与复习调度
> 如需参考历史决策，可继续阅读；如只需当前最佳实践，请参考 [{新条目编号}]。
```
